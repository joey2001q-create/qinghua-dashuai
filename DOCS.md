## 🔄 用户学习流程图（教育闭环）

```
    ┌──────────────────────────────────────────────┐
    │                                              │
    ▼                                              │
┌────────┐    ┌────────┐    ┌────────┐    ┌───────┴──┐
│ 设定目标│───▶│ 制定计划│───▶│ 执行训练│───▶│ 诊断反馈 │
│ ABC提分 │    │ 学习计划│    │ 考试冲刺│    │ 失分分析 │
│ 表      │    │        │    │ 靶向提分│    │ 卷后提分 │
└────────┘    └────────┘    └────────┘    └──────────┘
     ▲                                          │
     │              调整目标                     │
     └──────────────────────────────────────────┘

详细流程：
1. ABC目标提分表 → 设定各科A/B/C三级目标，评估信心指数
2. 学习计划 → 基于目标生成三阶段递进计划（基础→专项→冲刺）
3. 考试冲刺模块 → 执行训练（冲刺计划/题感训练/考点检测）
4. 靶向提分模块 → 深度学习（预习/费曼/作文）
5. 诊断反馈 → 失分分析/卷后提分 → 发现薄弱点
6. 回到步骤1 → 更新目标，调整信心指数，循环提升

辅助模块：
- 学习搭子 → 全程陪伴，随时答疑
- 你的树洞 → 心理疏导，情绪支持
```

## 📖 学习规划模块 (`/plan`)

### 1. ABC目标提分表 (`/plan/abc`)

| 功能 | 状态 | 说明 |
|------|------|------|
| 学段/年级选择 | ✅ | 小学/初中/高中分组，联动科目和满分默认值 |
| 科目联动 | ✅ | 小学(语数英)、初中(7科)、高中(9科) |
| 添加科目目标 | ✅ | 科目、满分、上次分、当前分、目标分、信心指数 |
| ABC目标计算 | ✅ | A=理想目标, B=预期可达(含边际递减), C=当前分数 |
| 信心指数锚定 | ✅ | 1-10级，每级有文字描述（1=完全没把握...10=志在必得） |
| 边际递减公式 | ✅ | 提分=差距×(信心/10)×(1-当前分/满分×0.5)，高分段提分更难 |
| 上次分数/趋势 | ✅ | 记录上次分数，自动计算趋势（↑上升/↓下降/→平稳） |
| 多科雷达图 | ✅ | 3科以上自动显示RadarChart对比C/B/A目标 |
| AI深度分析 | ✅ | 流式输出，含趋势分析/学习路径/心理建设/风险提示 |
| 导出Markdown | ✅ | 导出完整ABC目标表（含趋势和信心描述） |
| Markdown渲染 | ✅ | 表格、列表美化 |

**公式**：提分 = 差距 × (信心指数 / 10) × (1 - 当前分/满分 × 0.5)，B目标 = 当前分数 + 提分（含边际递减效应）

## 🚀 国内服务器部署

### 部署架构

```
用户 → Nginx (80/443) → PM2 → Next.js (3000)
                              ↓
                        GitHub Actions → 自动部署
```

### 部署文件

| 文件 | 说明 |
|------|------|
| `ecosystem.config.js` | PM2 进程管理配置 |
| `deploy/nginx.conf` | Nginx 反向代理配置 |
| `deploy/install.sh` | 一键部署脚本（Ubuntu/CentOS） |
| `deploy/update.sh` | 更新脚本 |
| `.github/workflows/deploy.yml` | GitHub Actions 自动部署 |

### 快速部署步骤

#### 1. 服务器准备（Ubuntu示例）

```bash
# 下载并执行一键部署脚本
curl -fsSL https://raw.githubusercontent.com/YOUR_USERNAME/qinghua-dashuai/main/deploy/install.sh | bash
```

#### 2.手动部署步骤

```bash
# 安装 Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo bash -
sudo apt install -y nodejs nginx

# 安装 PM2
sudo npm install -g pm2

# 克隆代码
sudo mkdir -p /var/www
cd /var/www
git clone https://github.com/YOUR_USERNAME/qinghua-dashuai.git
cd qinghua-dashuai

# 安装依赖并构建
npm install
npm run build

# 启动应用
pm2 start ecosystem.config.js
pm2 save
pm2 startup

# 配置 Nginx
sudo cp deploy/nginx.conf /etc/nginx/sites-available/qinghua-dashuai
sudo ln -s /etc/nginx/sites-available/qinghua-dashuai /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl restart nginx
```

#### 3.配置 GitHub Actions 自动部署

在 GitHub 仓库设置中添加 Secrets：

| Secret | 说明 |
|--------|------|
| `SERVER_HOST` | 服务器IP地址 |
| `SERVER_USER` | SSH用户名（如 root） |
| `SERVER_SSH_KEY` | SSH私钥 |
| `SERVER_PORT` | SSH端口（默认22） |

推送代码到 main 分支即可自动部署。

### 常用运维命令

```bash
# 查看应用状态
pm2 status

# 查看日志
pm2 logs qinghua-dashuai

# 重启应用
pm2 restart qinghua-dashuai

# 手动更新
cd /var/www/qinghua-dashuai && git pull && npm run build && pm2 restart qinghua-dashuai

# 查看 Nginx 状态
sudo systemctl status nginx

# 测试 Nginx 配置
sudo nginx -t
```

### 环境变量配置

创建 `/var/www/qinghua-dashuai/.env.local`：

```env
# AI API配置（如需更换）
NEXT_PUBLIC_API_URL=https://your-api-endpoint.com

# OCR服务地址
PADDLE_OCR_URL=http://localhost:8868/predict
```

### 性能优化建议

| 配置项 | 建议值 | 说明 |
|--------|--------|------|
| PM2 instances | 2-4 | 根据CPU核心数调整 |
| max_memory_restart | 500M | 内存超限自动重启 |
| Nginx worker_processes | auto | 自动匹配CPU核心数 |
| gzip | 开启 | 减少传输体积 |

```css
/* 背景层级 */
页面背景: bg-slate-900

分割线: border-slate-700

/* 圆角 */
卡片: rounded-2xl
按钮/输入框: rounded-lg
标签: rounded-md 或 rounded

/* 文字颜色层级 */
标题: text-white
正文: text-slate-300
辅助说明: text-slate-400
占位符: text-slate-500
强调色: text-indigo-400 / text-emerald-400 / text-amber-400 / text-rose-400

/* 间距 */
卡片内边距: p-6
元素间距: space-y-4
网格间距: gap-4 或 gap-6
```
